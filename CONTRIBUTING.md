# CONTRIBUTING

Короткие правила разработки для этого репозитория.

## Обязательные правила

1. Комментарии в коде:
- Добавляем короткие комментарии простым языком (по возможности на русском), когда логика неочевидна.
- Не пишем комментарии на каждую строку; только там, где это реально помогает понять код быстрее.

2. Тесты при изменениях:
- Любой новый функционал должен идти с тестами.
- Любой рефакторинг, который затрагивает поведение, должен обновлять/добавлять тесты.
- Если тестов нет, сначала добавляем минимально достаточные тесты и только потом продолжаем рефакторинг.

3. Перед завершением задачи запускаем проверки:
- Frontend: `cd gamesales-web && npm run check`
- Backend: `cd api && make test-pytest` (или `make test-v`, если pytest недоступен)

## Чеклист перед merge

- [ ] Код читаемый, сложные места прокомментированы.
- [ ] Тесты добавлены/обновлены под изменения.
- [ ] Все локальные проверки проходят.
